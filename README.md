# Project Overview

This project is divided into two main parts: the LLM (Large Language Model) approach and the ML (Machine Learning) approach. Each part has its own respective Python file that can be executed independently.

The project utilizes Docker for containerization, ensuring consistent and reproducible environments. This README will guide you through the steps to run each part of the project using Docker, as well as describe the project folder structure.

## Folder Structure

- **data**: Contains the datasets used for training and testing.
  - `cleaned_test_data.tsv`: Cleaned test dataset for LLM  model.
  - `cleaned_train_data.tsv`: Cleaned training dataset for LLM  model.
  - `cleaned_test_data_ml.tsv`: Cleaned test dataset specifically for the ML model.
  - `cleaned_train_data_ml.tsv`: Cleaned training dataset specifically for the ML model.
  - `TWNERTC_TC_Fine_Grained...`: Original dataset used in preprocessing.

- **models**: Contains the scripts for the LLM and ML models.
  - `llm_model.py`: Python script for executing the LLM-based model.
  - `ml_model.py`: Python script for executing the ML-based model.

- **preprocessing**: Contains scripts used to preprocess datasets.
  - `dataset_ml.py`: Preprocessing script used specifically for ML datasets.
  - `dataset.py`: Preprocessing script for LLM model datasets purposes.

- **results**: Contains the output from the models.
  - `llm`: Folder containing results generated by the LLM model.
  - `ml`: Folder containing results generated by the ML model.

- **Dockerfile**: Used to build the Docker container.
- **README.md**: This file.
- **requirements.txt**: Contains all dependencies for the project.

## How to Run the Project

### Prerequisites
- Docker should be installed on your system.

### Build the Docker Image

To build the Docker image, run the following command in your terminal from the root folder of the project:

```bash
docker build -t hepsiburadacasestudy .
```

### Running the LLM Model

To execute the `llm_model.py` script inside the Docker container, use the following command:

```bash
docker run -v $(pwd)/data:/app/data -v $(pwd)/results/llm:/app/results/llm hepsiburadacasestudy python models/llm_model.py
```
**Note**: The `llm_model.py` script may not work as intended because it relies on accessing the Ollama API and LLM model, which were used locally during development but could not be bound to the Docker container due to computational errors. You may encounter issues with accessing these resources when running the script within the container.
This command will mount the `data` folder and the `results/llm` folder to the container so that you can use the datasets and store the output.

### Running the ML Model

To execute the `ml_model.py` script inside the Docker container, use the following command:

```bash
docker run -v $(pwd)/data:/app/data -v $(pwd)/results/ml:/app/results/ml hepsiburadacasestudy python models/ml_model.py
```

This command will mount the `data` folder and the `results/ml` folder to the container so that you can use the datasets and store the output.

## Folder Mounts Explained
- `-v $(pwd)/data:/app/data`: This mounts the `data` directory from your host machine to the Docker container at `/app/data`.
- `-v $(pwd)/results/llm:/app/results/llm`: This mounts the `results/llm` directory from your host machine to the Docker container at `/app/results/llm`. This allows you to save the output from the LLM model to your local machine.
- `-v $(pwd)/results/ml:/app/results/ml`: This mounts the `results/ml` directory from your host machine to the Docker container at `/app/results/ml`. This allows you to save the output from the ML model to your local machine.

## Requirements
The `requirements.txt` file contains all the necessary Python libraries for the project. These dependencies are installed during the Docker build process.

## Summary
- The project is split into two parts, each executed from a different Python script: `llm_model.py` for the LLM-based approach and `ml_model.py` for the ML-based approach.
- Use Docker to build and run the project with the provided instructions.
- Results are stored in their respective folders under `results/`.
